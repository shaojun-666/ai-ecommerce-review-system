## 2026-06-10 — Phase 2: JD.com 爬虫引擎 (完成)

### 完成工作

#### 反反爬中间件 (AntiBot)
- 创建 `AntiBotMiddleware`：UA 轮换（9 个真实浏览器 UA）、令牌桶频率限制（1-3s 随机抖动）、Cookie 会话维持
- CAPTCHA/WAF 检测：检查 HTTP 403/429 状态码 + 响应内容签名（captcha、验证码、WAF 等 20+ 模式）
- 短 HTML 检测：小于 2000 字节的 HTML 页面（CAPTCHA 页面特征）

#### 爬虫基类 (策略模式)
- 创建 `BaseCrawler` 抽象基类：`extract_product()`、`extract_reviews()`、`get_review_url()` 抽象方法
- 6 种异常类型：CrawlerError、CrawlerTimeoutError、CrawlerBlockedError、EmptyResponseError、ExtractionError、PageNotFoundError
- 指数退避重试（基础 2s，最大 3 次），遇 404 直接抛出（不重试）
- `crawl_all`：聚合商品 + 多页评论，遇到错误停止并返回已收集数据

#### JD.com 适配器
- 商品详情：BS4 解析 item.jd.com HTML（sku-name、p-price、spec-img、shop-name）
- 评论 API：club.jd.com XHR JSONP 接口，`_clean_jsonp()` 去除回调包装
- 产品 ID 提取：正则从 item.jd.com URL 提取数字 ID
- JSON 解析保护：处理空评论页、无效评分（默认 0）、空内容过滤

#### Celery 任务
- `run_crawl`：执行爬取 → upsert Product → 批量插入 Comment（50/批，content 去重）
- `check_crawl_timeouts`：标记运行超过 1 小时的任务为失败（Celery Beat 每 5 分钟）
- `schedule_due_crawls`：启动到期的定时爬取任务（Celery Beat 每 60 秒）

#### REST API
- `GET/POST /crawl/tasks` — 创建/列表爬虫任务
- `GET /crawl/tasks/<id>` — 任务详情
- `POST /crawl/tasks/<id>/start` — 手动触发
- `PUT/DELETE /crawl/tasks/<id>` — 更新/删除
- `GET /crawl/stats` — 聚合统计
- JWT 认证保护，URL 白名单限制（仅 item.jd.com / club.jd.com）

#### 修复记录
- **Chinese byte literals**：`_BLOCK_SIGNATURES` 中 `b"验证码"` 改为字符串，通过 `response.text` 比较
- **Dict comprehension bug**：`[{"key": val for i in range(10)}]` → `[{"key": val} for i in range(10)]`（列表推导 vs 字典推导）
- **fail() syntax errors**：15 处 `return fail"..." )` 改为 `return fail("...")`
- **jwt_required import**：错误导入名 `jwt_required` → `require_auth`
- **purchase_time 类型转换**：字符串日期 → datetime.date（SQLite DateTime 约束）
- **Celery request 属性**：移除 `patch.object(run_crawl, "request")`（Celery property 没有 setter）
