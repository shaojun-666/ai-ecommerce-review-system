### 完成工作

#### P11-1: 定时邮件报告
- SMTP 配置集成：`SMTP_SERVER/PORT/USERNAME/PASSWORD/USE_TLS/MAIL_FROM/REPORT_RECIPIENTS`

#### P11-2: 警报系统
- **Alert 模型**：types(`negative_surge|price_drop|price_spike|new_competitor|crawl_failure`), severities(`critical|warning|info`)
- **3 个检测器**：
  - `negative_surge` — 7 日内负面评论超前期 1.5 倍（最少 3 条）
  - `price_anomaly` — 30 日内价格波动 ≤-15%(price_drop) 或 ≥+20%(price_spike)
  - `silent_products` — 14 天无新评论的商品
- **24h 去重**：同 product + type 24 小时内不重复创建
- **REST API**：列表、标记已读、标记全部已读、手动触发检测、未读数
- **前端 AlertList.vue**：严重度标签、未读/全部筛选、单条标记、批量操作
- **Header 徽章**：AppLayout 右上角实时未读计数

#### P11-3: WebSocket 实时推送
- Flask-SocketIO + gevent 异步模式，`/ws` 命名空间
- 事件：`connect/disconnect/ping`、`broadcast_dashboard_update`、`broadcast_alert_notification`
- **前端 socket.ts**：自动检测 WS URL，`websocket` 优先 + `polling` 降级
- Dashboard：SocketIO 监听 `dashboard_update` 事件，降级时 60s 轮询
- Alert 徽章：SocketIO 实时推送 `alert_count`，动态更新未读计数
- nginx WebSocket 代理：Upgrade 头部转发，3600s 超时

#### P11-4: 竞品自动发现
- **4 因子评分模型**：标签重叠(40pts) + 同平台(20pts) + 名称关键词(30pts) + ID 前缀(10pts)，满分 100
- `discover_competitors(product_id)` — 单商品竞品发现
- `discover_all_competitor_groups()` — 全库成对聚类
- REST API：`GET /competitors` + `GET /competitors/summary`

#### P11-5: HTTPS + 生产环境加固
- **nginx 双模式**：生产模式(443 SSL + 80→301 跳转) + 开发模式(80 HTTP)
- 安全头部：X-Content-Type-Options、X-Frame-Options、X-XSS-Protection、Referrer-Policy、Permissions-Policy、HSTS
- Gzip 压缩：text/css/json/javascript/svg+xml，最小 1024 字节
- 静态缓存：30d expires，`public, immutable`
- 隐藏文件拦截：`deny all` 屏蔽 `.git`、`.env` 等
- **后端中间件**：`after_request` 自动添加安全头部
- **日志轮转**：logrotate 配置，14 天轮转，压缩

## 2026-06-10 — Phase 3: 数据管道 (完成)

### 完成工作

#### 文本清洗工具 (text_cleaner)
- 创建 `backend/app/utils/text_cleaner.py`：5 个函数覆盖清洗全流程
- `clean_text()`：HTML 标签剥离 → URL 移除 → 空白折叠 → 重复字符截断（先剥 HTML 再除 URL，避免属性内 URL 残留）
- `normalize_rating()`：`int(x + 0.5)` 代替 `round()`，避免 Python 3 银行家舍入（`round(4.5) == 4`）
- `is_valid_review()`：长度 5-10000 字符、纯标点检测
- `detect_language()`：中文字符占比 >30% 判定为中文
- `content_hash()`：SHA-256 清洗后文本的十六进制摘要

#### 数据管道服务 (DataPipeline)
- 创建 `backend/app/services/data_pipeline.py`：集中式清洗/校验/去重/规范化
- `process_review()`：单条评论处理，返回 `ProcessedReview` 标准化对象
- `process_batch()`：批量处理，内存去重 + DB 级哈希去重
- `filter_existing_hashes()`：按 `(content_hash, product_id)` 查询已存在的记录
- 支持中英文双栏位名（`content` / `评论内容`、`rating` / `评分` 等）
- purchase_time 字符串 → date 自动转换

#### Comment 模型扩展
- 新增 `content_hash` 字段（`db.String(64)`, indexed）
- 新增复合索引 `idx_comments_hash_product(content_hash, product_id)` 支持高效去重查询
- 创建 Alembic 迁移脚本 `a1b2c3d4e5f6_add_content_hash_to_comments.py`

#### 集成到所有导入路径

**爬虫 (crawl_tasks.py)**：DataPipeline 替代内联去重
- 先过管道进行清洗 + 校验 + 规范化
- DB 级 content_hash 去重替代 `seen_contents` 内存集合
- 过滤统计日志（total / dup / filtered）

**CSV 导入 (comment_service.py)**：
- 三阶段流程：逐行校验（保留行号用于错误报告）→ DB 哈希去重 → 批量插入
- content_hash 自动填充，重复行正确跳过

**分析前清洗 (analysis_tasks.py)**：
- 评论内容先过 `clean_text()` 再送入 BERT 模型
- 去除 HTML 标签和 URL 等噪声，提升推理质量

#### 修复记录
- **URL 正则太贪婪**：`https?://\S+` 会吞噬 `">链接</a>` 等后续内容，留下 `href="` 悬空标签。改为 `https?://[^\s<>"']+` 并在 URL 去除前先剥 HTML
- **rating=0 被当作缺失**：`review.get("rating") or review.get("评分")` 中 0 是 falsy，导致有效评分 0 被跳过。改用显式 `is None` 检查
- **银行家舍入问题**：`round(4.5) == 4` 不符合评分直觉。改用 `int(x + 0.5)` 实现四舍五入

- 创建 `AntiBotMiddleware`：UA 轮换（9 个真实浏览器 UA）、令牌桶频率限制（1-3s 随机抖动）、Cookie 会话维持

- 创建 `BaseCrawler` 抽象基类：`extract_product()`、`extract_reviews()`、`get_review_url()` 抽象方法
- 6 种异常类型：CrawlerError、CrawlerTimeoutError、CrawlerBlockedError、EmptyResponseError、ExtractionError、PageNotFoundError
- 指数退避重试（基础 2s，最大 3 次），遇 404 直接抛出（不重试）
- `crawl_all`：聚合商品 + 多页评论，遇到错误停止并返回已收集数据

#### JD.com 适配器
- 商品详情：BS4 解析 item.jd.com HTML（sku-name、p-price、spec-img、shop-name）
